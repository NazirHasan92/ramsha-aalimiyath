## Per-chapter deliverables

Every completed chapter has four matching artefacts. From the deep-notes page of any chapter, a banner at the top jumps to the other three formats.

<div class="grid cards" markdown>

-   :material-notebook-edit-outline:{ .lg .middle } &nbsp; **Deep notes** *(Markdown)*

    ---

    Personal learning material — Part A (understand the chapter) followed by Part B (when ready, teach it back). Renders on this site with the Material theme.

-   :material-school-outline:{ .lg .middle } &nbsp; **Teacher view** *(HTML)*

    ---

    Structured chapter for teaching the cohort. Custom CSS designed for screen + projector. Opens as a standalone HTML page.

-   :material-file-document-multiple-outline:{ .lg .middle } &nbsp; **Student handout** *(HTML, print-friendly)*

    ---

    Printable / shareable summary the cohort takes home. Optimised for paper.

-   :material-presentation-play:{ .lg .middle } &nbsp; **Slides** *(HTML presentation)*

    ---

    Slideshow for classroom delivery — section-by-section walkthrough.

</div>

## About this project

<div class="grid" markdown>

!!! info "Student-teacher method"
    Each chapter is **first learned thoroughly, then re-taught** to the cohort. The 4 deliverables emerge in that order: deep-notes (for Ramsha's own understanding) → teacher-notes (to deliver) → student-handout (to share) → slides (to project).

!!! check "Source-only"
    Every ruling, hadith, and verse traces only to its source PDF. **No outside fiqh sources, no general-knowledge fills, no AI-generated Islamic content.** Validators (`fiqh-validator`, `hadeeth-validator`, `arabic-validator`) catch drift.

!!! example "Companion site"
    Nazir's Nahw + Sarf notes at [eMahad-aalimiyath-site](https://nazirhasan92.github.io/eMahad-aalimiyath-site/) — same eMahad cohort, different subjects.

</div>
